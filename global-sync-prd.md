# PRD: Global Sync — 자막 자동 생성 & 번역 스튜디오

**문서 버전:** v1.0
**작성일:** 2026.03.24
**제품:** FanMaker — 창작 스튜디오 > 글로벌 싱크

---

## 1. 개요 (Overview)

### 배경

FanMaker는 K-pop 팬들이 창작물을 만드는 플랫폼이다. 현재 글로벌 싱크는 3단계(영상 소스 → 번역 설정 → 자막 편집) UI 껍데기만 존재하며 실제 자막 생성 기능이 동작하지 않는다. K-pop 팬덤은 본질적으로 글로벌하며, 한국어 영상을 영어·일본어·스페인어 등으로 번역한 자막을 만드는 수요가 매우 크다. 이 기능이 실제로 동작하면 팬들이 직접 자막을 만들어 공유할 수 있는 강력한 차별화 포인트가 된다.

### 목표

YouTube URL을 붙여넣으면 원하는 언어의 자막이 자동으로 생성되고, 유저가 이를 편집 후 SRT 파일로 다운로드하거나 팬 유니버스에 공유할 수 있도록 한다. **비용 0원** 기술 스택으로 구현한다.

### 핵심 지표 (Success Metrics)

자막 생성 완료율 80% 이상(URL 입력 후 자막 출력까지), 평균 처리 시간 30초 이내, 생성된 자막의 팬 유니버스 공유 전환율 30% 이상을 목표로 한다.

---

## 2. 유저 스토리

**메인 시나리오:** 일본인 팬 유저가 BTS 뮤직비디오 YouTube URL을 붙여넣고, 번역 언어로 "일본어"를 선택한다. 30초 안에 타임스탬프가 붙은 일본어 자막이 생성되고, 유저는 어색한 표현 몇 군데를 수정한 뒤 SRT 파일로 다운로드한다.

**엣지 케이스 시나리오 1:** 자막이 비활성화된 영상 → 오디오 추출 후 Whisper 로컬 처리로 fallback, 처리 중임을 유저에게 진행 바로 표시한다.

**엣지 케이스 시나리오 2:** 15분 이상 긴 영상 → 청크 단위로 분할 처리하며 실시간으로 자막 줄이 쌓이는 스트리밍 UX를 보여준다.

---

## 3. 기능 요구사항

### Step 1 — 영상 소스 (현재 존재, 기능 구현 필요)

YouTube URL 입력 시 실시간으로 영상 썸네일, 제목, 길이를 미리보기로 보여준다. URL 유효성 검사(유튜브 영상이 맞는지, 비공개/삭제 영상 여부)를 즉시 수행한다. MP4 업로드는 100MB 이하 파일을 지원한다. "다음 단계" 버튼은 유효한 소스가 확인된 후에만 활성화된다.

### Step 2 — 번역 설정 (현재 존재, 기능 구현 필요)

원본 언어는 자동 감지를 기본으로 하되 수동 선택도 허용한다. 타겟 언어 선택기는 우선 지원 언어 10개(영어, 일본어, 중국어 간체, 중국어 번체, 스페인어, 프랑스어, 인도네시아어, 태국어, 베트남어, 포르투갈어)를 제공한다. 자막 스타일 옵션으로 번역만 / 원문+번역 병기 두 가지를 선택할 수 있다. 처리 시작 전에 예상 소요 시간을 표시한다.

### Step 3 — 자막 편집기 (현재 존재, 기능 구현 필요)

좌측에 YouTube IFrame 플레이어를 임베드하고, 우측에 타임스탬프별 자막 라인 목록을 보여준다. 플레이어 재생 시간에 맞춰 현재 자막 라인이 하이라이트된다. 각 자막 라인을 클릭하면 인라인으로 텍스트 편집이 가능하다. 영상 위에는 `position: absolute`로 자막 오버레이를 렌더링하여 실시간 확인이 가능하다. 완성된 자막은 SRT, VTT 형식으로 다운로드할 수 있고, 팬 유니버스에 "글로벌 싱크" 태그로 공유할 수 있다.

## 4. 기술 스펙

### 전체 파이프라인

```
[유저] YouTube URL 입력
       ↓
[프론트] URL 유효성 확인 → 썸네일/제목 미리보기 표시
       ↓
[백엔드 API] /api/subtitle/generate 호출
       ↓
  ┌────────────────────────────────┐
  │ [경로 A] YouTube 자막 추출      │  ← 자막 있는 영상 (90% 케이스)
  │ youtube-transcript-api (Python)│
  │ → 원문 자막 + 타임스탬프 반환   │
  └────────────────────────────────┘
       ↓ 자막 없을 경우 fallback
  ┌────────────────────────────────┐
  │ [경로 B] 오디오 → STT          │  ← 자막 없는 영상 (10% 케이스)
  │ yt-dlp 오디오 추출             │
  │ → Whisper 오픈소스 로컬 실행   │
  │ → 타임스탬프 포함 자막 생성    │
  └────────────────────────────────┘
       ↓
[번역] DeepL Free API (월 50만 자)
       ↓ 한도 초과 시 fallback
       deep-translator (Google 비공식)
       ↓
[응답] SRT 형식 자막 데이터 반환
       ↓
[프론트] 자막 편집기 렌더링
```

### 백엔드 스택

Python FastAPI 서버를 Railway 무료 플랜에 배포한다. `youtube-transcript-api` 라이브러리로 YouTube 자막을 직접 가져온다. Whisper 오픈소스 모델(`whisper-small`, 약 244MB)을 서버에 올려두고 fallback으로 사용한다. 번역은 DeepL Free API를 1순위, `deep-translator`(Google 비공식 래핑)를 2순위 fallback으로 사용한다.

### 프론트엔드 스택

기존 Next.js 프로젝트에 YouTube IFrame Player API를 연동한다. 자막 오버레이는 플레이어 위에 `position: absolute` div로 구현하고, `setInterval`로 100ms마다 `player.getCurrentTime()`을 폴링하여 현재 시간에 맞는 자막 라인을 표시한다. 자막 편집기는 좌우 분할 레이아웃으로, 우측 라인 클릭 시 플레이어가 해당 타임스탬프로 점프한다.

### API 설계

```
POST /api/subtitle/generate
Body:     { url: string, targetLang: string, style: "translated" | "bilingual" }
Response: { subtitles: [{ start: number, end: number, original: string, translated: string }] }

GET /api/subtitle/status/:jobId
Response: { status: "pending" | "processing" | "done" | "error", progress: number }
```

긴 영상(5분 이상)은 비동기 Job 방식으로 처리하고, 프론트는 SSE(Server-Sent Events) 또는 폴링으로 진행 상황을 실시간으로 받아 표시한다.

## 5. UX 흐름 상세

### 처리 중 상태

"다음 단계" 클릭 후 처리 중에는 Step 2 화면에서 단계별 진행 상태를 텍스트로 표시한다.

```
✓ 영상 정보 확인 중...
✓ 자막 데이터 가져오는 중...
✓ 한국어 → 일본어 번역 중... (87%)
✓ 완료!
```

예상 대기 시간도 함께 표시한다.

### 에러 처리

비공개/삭제 영상이면 "해당 영상은 접근할 수 없습니다" 인라인 에러를 URL 입력 필드 아래에 표시한다. 번역 API 한도 초과 시 자동으로 fallback 번역을 수행하며 유저에게는 별도 알림 없이 처리한다. Whisper fallback 시에는 "자막이 없는 영상이라 오디오 분석을 진행합니다. 약 2~3분 소요됩니다"를 안내한다.

### 자막 편집기 UX

편집기에서 라인을 클릭하면 해당 시점부터 영상이 자동 재생되어 문맥을 바로 확인할 수 있다. 편집 내용은 자동 저장(debounce 1초)된다. 영상 오버레이 자막 폰트 크기, 위치는 편집기 우측 상단 설정 버튼으로 조절 가능하다. 자막 라인 추가/삭제/타임스탬프 조정도 편집기에서 가능하다.

## 7. 제약사항 및 한계

DeepL Free 월 50만 자 한도는 약 1,000개의 3~4분짜리 영상 처리 분량이다. 한도 초과 시 Google 비공식 API로 자동 전환되며 품질 차이가 발생할 수 있다. Whisper 로컬 실행은 CPU 환경 기준 1분 영상에 약 30~60초가 소요된다. Railway 무료 플랜의 월 500시간 제한을 고려해 슬립 모드 진입을 방지하는 헬스체크 핑을 설정한다. YouTube Terms of Service상 자막 데이터 추출은 개인적, 비상업적 용도에서 허용되는 그레이존이므로 이용약관에 명시가 필요하다.

## 9. 미결 사항 (Open Questions)

생성된 자막을 팬 유니버스에 공유할 때 원본 YouTube 영상과 연결하는 방식(임베드 vs 링크)을 결정해야 한다. Pro 유저에게는 더 높은 번역 품질(DeepL Pro)을 제공하는 유료 차별화 방안을 검토할 필요가 있다. 자막 저장을 위한 DB 스키마 설계(현재 FanMaker DB 구조 확인 필요)도 별도로 논의해야 한다.
